Instance Resource Allocation

It is important to allocate the correct resources to each instance. By default, all instances are allocated 4G of RAM. This can be changed in the configuration file, however per-instance configurations may need to be made as well.

It is important to consider how the instance will be used. Several factors will affect performance:

  • How many players will be online at any one time.
  • What these players might be doing (I.E: Lots of redstone, etc.)
  • Where these players are (if everyone's in different locations, more of the world will be loaded at once).

Minecraft is (mostly) single-threaded, so CPU clock speed matters more than cores. As long as you have any recent quad-core processor with a decent clock speed, you should be okay in the CPU department. Examples of good Minecraft processors would be a recent Core i5, Core i7, Ryzen 3, or Ryzen 5.

Please read: For the best performance, use Paper MC. It is much more efficient than the vanilla Minecraft server JAR. Gorp is created for and uses Paper MC. You don't have to change any settings to get the benefits of Paper MC. Should you use a custom server JAR, please consider the performance of the JAR you are using and follow the developer's recommendations.

This is my personal guideline that has worked well for me:

Player Count View Distance RAM Allocated
1-5 15 chunks 2 GB
6-10 15 chunks 5 GB
11-20 15 chunks 10 GB

This was on a machine with 16 GB total RAM and an Intel Core i7-6700K processor (on the older side, but still plenty for Minecraft).
