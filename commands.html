Gorp Command Reference

The command structure of Gorp is: gorp [action] [arguments]...

Items with a question mark in them are optional. In gorp create-world [server-name] [?world-name], the world name is optional.

Start & Stop Instances see: start & stop

Command Description
gorp start [server-name] Starts an instance.
gorp stop [server-name] [?arguments] Stops an instance. By default, takes a backup and gives a warning.
Arguments accepted:
  • fast: Skips the backup
  • now: Skips both backup and warning. Immediate shutdown.
  • power: Same as a fast shutdown, however the warning specifies the reason for the shutdown is due to a power outage.
gorp restart [server-name] [?arguments] Restarts an instance. By default, takes a backup and gives a warning.
Arguments accepted:
  • fast: Skips the backup
  • now: Skips both backup and warning. Immediate restart.

Create & Delete Instances see: manage instances

Command Description
gorp create-server [server-name] Creates a new server instance with the passed name.
The instance is created in /minecraft/servers/[name].
gorp delete-server [server-name] Deletes the given server instance's files. This action is not reversible. Be sure to take a backup and double-check which instance you want to delete.

Manage Worlds see: manage worlds

Command Description
gorp create-world [server-name] [?world-name] Creates a world with the given name in the specified server. The world will be created next time the server instance is started. Optionally pass the world name to avoid a prompt.
gorp delete-world [server-name] [?world-name] Delets a world with the given name in the specified server. This action is not reversible. Be sure to take a backup and double-check which world you want to delete. Optionally pass the world name to avoid a prompt.
gorp archive-world [server-name] [?world-name] Compresses the specified world and moves it to the archives destination. The world is then removed from the server instance's files to reduce clutter.
gorp switch-world [server-name] [?world-name] Switch the world files the server will use when it starts next. Optionally pass the world name instead of picking it from a list.

Backup & Restore Worlds see: backups

Command Description
gorp backup-world [server-name] Backs up the world files for the given server instance. The currently active world will be the one to be backed up. Learn more about this behavior.
gorp restore-world [server-name] Presents you with options of what world you wish to restore, along with specifying the backup's date and exact file.

Update & Download Server JARs see: jar updater

Command Description
gorp update-jar Download the latest stable Paper JAR file, install it, and set it as the default JAR file.
gorp get-jar [game version] Download a Paper JAR file for a specified Minecraft game version. Only supports certain versions.
View Supported Game Versions

Update Gorp see: gorp self-upgrade

Command Description
gorp upgrade Upgrades the installation of Gorp to the latest available version.
Caution: Updating Gorp will overwrite all shell script files and the run.sh file, except for where changes are anticipated.
