Starting & Stopping Instances

Start an Instance

sudo gorp start [name]

When an instance is started, a new screen session of the same name is started. Then, the server's run.sh script is executed within. This allows the Minecraft server to begin starting up in the background.

Stopping an Instance

sudo gorp stop [name] [options]

When an instance is stopped, a backup of the world files are taken, a 30 second warning is sent to players in-game, and then the server is gracefully shut down.

Shutdown options:

  • fast: Skip taking a backup, only give a warning. sudo gorp stop [name] fast
  • now: Immediately shut the server down (no warning). sudo gorp stop [name] now

Restarting a Server

sudo gorp restart [name] [options]

When an instance is restarted, a backup of the world files are taken, a 30 second warning is sent to players in-game, and then the server is gracefully shut down and brought back online.

Restart options:

  • fast: Skip taking a backup, only give a warning. sudo gorp restart [name] fast
  • now: Immediately restart the server (no warning). sudo gorp restart [name] now

Access a Running Instance's Console

sudo screen -r [name]

Gorp uses the fantastic Screen utility to execute a server instance in the background. Screen creates virtual sessions that can be resumed at any time. When a server is started, Gorp puts the instance in a Screen session of the same name as the instance.

To disconnect from the Screen session: CTRL + A + D
