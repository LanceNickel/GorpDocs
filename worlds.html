Creating, Deleting, Archiving, & Switching Worlds

Create a World

sudo gorp create-world [server-name] [?world-name]

This will update the server instance's configuration, generate the world, and create an intitial world backup. Optionally pass a name for the world to skip an additional prompt.

World names may NOT have spaces or special characters in them.

Delete a World

sudo gorp delete-world [server-name] [?world-name]

This deletes the world in the instance you specified. This action is not recoverable, please take a backup! Optionally pass a name for the world to skip an additional prompt.

Archive a World

sudo gorp archive-world [server-name] [?world-name]

This archives the world in the instance you specified. Heads up: An archived world must be restored manually. This is intended to send world archives to one place for systematic long-term storage. Optionally pass a name for the world to skip an additional prompt.

Switch Worlds

sudo gorp switch-world [server-name] [?world-name]

Update a server instance's configuration to another world directory in the instance. Optionally pass a name to avoid needing to choose the world from a list.
